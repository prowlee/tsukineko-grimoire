# My Shelf — 表示・整理のロードマップ

Notion 的な「整理された一覧」を段階的に足すための指針です。

## 進め方の原則

1. **スキーマを増やす前に UI を固める** — まず表・ソートで満足度を上げ、必要になったら Firestore にフィールド追加。
2. **モバイルは別レイアウト** — 表は `md:` 以上のみ。小画面はカードのまま（可読性・タップ領域）。
3. **依存を増やしすぎない** — Phase A/B はネイティブ `<table>` + `useMemo`。件数が数百超えたら TanStack Table / 仮想スクロールを検討。

## フェーズ

| Phase | 内容 | スキーマ |
|-------|------|----------|
| **A（済）** | デスクトップ表一覧 + クライアントソート（追加日・公開日・タイトル） | 変更なし |
| **B（済）** | 列の表示切替、フィルタ（カテゴリ・タグ OR） | 変更なし |
| **C（済）** | メモ・読了状態（未読/読書中/読了）・マイタグ、PATCH API、フィルタ拡張 | `memo`, `readStatus`, `userTags` |
| **D（済）** | カンバン（列間ドラッグで状態更新）、表示設定の localStorage 永続化（`tg-shelf-prefs-v1:{uid}`） | Firestore の `viewPrefs` は未使用（ブラウザのみ） |

## 関連ファイル

- `app/api/shelf/route.ts` — 追加時の非正規化フィールド

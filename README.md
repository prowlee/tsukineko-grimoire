# Tsukineko Grimoire（月ねこグリモワール）

**用易懂的语言捕捉AI·机器学习论文，留存手边，随时回听。**  
这是一款作为个人“知识之书”来培养的应用。

---

## 问题 → 解决方案

论文晦涩难懂，且多为英文，容易让人望而却步。  
即使读了，也难以将其应用于工作或学习。

这款应用旨在降低这一障碍。  
它将原本艰深的信息，**转化为你能理解的知识**，  
并将“查找、积累、回听”整合为一个流程。

---

## 适合人群

- 刚开始学习AI的人
- 曾尝试阅读论文但未能坚持的人
- 希望在实务或策划中运用基于论文的知识的人
- 希望将感兴趣的文献进行个人化整理和再利用的人

---

## 核心体验（4项）

### 查找

通过主题或关键词，找到相关的论文。  
可以在列表中浏览，轻松挑选出接下来要读的候选。

### 理解

便于用中文把握论文标题和摘要的内容，  
无需仅依赖冗长的英文即可理解文章大意。

### 积累

将喜欢的论文像书架一样保存下来，  
通过笔记和状态管理，将其培养成“个人阅读清单”。

### 对话

以积累的知识为基础，通过文字提出疑问并进行深入探究。  
可以从引用内容直接返回原论文。

---

## 特点

- **以 arXiv 为中心** — 以公开论文为基础，便于在同一平台收集
- **用中文理解** — 降低开始阅读的门槛
- **保存与再利用** — 可以随时找回曾经发现的论文
- **通过聊天深入探究** — 阅读后也可通过问答进行整理

## 技术（面向开发者）

基于 Next.js / TypeScript、Firebase、Google Cloud（存储与搜索基础设施）。  
AI 搜索与回答仅通过 **Vertex AI Agent Builder（Discovery Engine）** 实现（使用 `@google-cloud/discoveryengine`）。详细信息请参阅 `PRD.md` 和 `.cursorrules`。

### 快速设置

```bash
cp .env.local.example .env.local
# 将 Firebase / GCP / Agent Builder 的配置值填入 .env.local

npm install
WATCHPACK_POLLING=true npm run dev -- --port 3002
```

数据导入示例：`bash scripts/curated-papers.sh 3002`（在开发服务器启动后执行）  
生产环境部署：使用 Cloud Run（请参阅 `.github/workflows/deploy.yml`）

详细规格、API、管理员使用的 curl 命令以及设计笔记均汇总于 **`PRD.md`**。
